# Quokka Smile Project — Design Guide

## 1. 프로젝트 개요

**Quokka Smile Project**는 쿼카의 따뜻하고 웃는 듯한 이미지를 통해 행복, 힐링, 생태 보호의 메시지를 전달하는 브랜드형 소개 웹사이트다.

이 페이지는 단순한 동물 소개에 머무르지 않고 다음 흐름을 만든다.

1. 쿼카의 귀여움으로 관심을 끈다.
2. 행복과 긍정의 이미지를 전달한다.
3. 서식지와 생태 보호의 필요성을 알린다.
4. 공유, 캠페인, 굿즈 페이지 등 다음 행동으로 연결한다.

---

## 2. 디자인 목표

### 핵심 키워드

- 따뜻한
- 귀여운
- 희망적인
- 부드러운
- 친근한

### 사용자가 느껴야 하는 인상

- 첫 화면에서 밝고 편안한 분위기가 느껴져야 한다.
- 쿼카가 프로젝트의 중심 캐릭터로 명확하게 보여야 한다.
- 정보가 어렵거나 무겁게 느껴지지 않아야 한다.
- 보호 메시지는 강요하기보다 자연스럽게 행동을 유도해야 한다.

---

## 3. 브랜드 메시지

### 메인 메시지

> 세상에 행복을 전하는 작은 미소, 쿼카

### 보조 메시지

> 행복은 쿼카처럼 작고 따뜻하게 번집니다.

### 콘텐츠 작성 원칙

- 짧고 따뜻한 문장을 사용한다.
- 전문적인 생태 정보도 초보자가 이해하기 쉽게 표현한다.
- 귀여움만 강조하지 않고 보호와 책임의 메시지를 함께 담는다.
- 과장되거나 검증되지 않은 생태 정보를 단정하지 않는다.

## 4. 컬러 시스템

현재 페이지의 색상은 CSS 사용자 정의 속성으로 관리한다.

| 토큰 | 색상 | 용도 |
|---|---:|---|
| `--yellow` | `#FFD86B` | 핵심 강조, 햇빛, 배지, 아이콘 배경 |
| `--yellow-light` | `#FFF1AF` | 연한 강조 배경, 그라디언트 |
| `--brown` | `#A9784B` | 브랜드 메인 컬러, 제목, 버튼 |
| `--brown-dark` | `#4B3423` | 본문 기본 글자색, 진한 배경 |
| `--cream` | `#FFF7E8` | 전체 페이지 배경 |
| `--white` | `#FFFFFF` | 카드, 버튼, 메뉴 배경 |

### 색상 사용 원칙

- 페이지 전체 배경은 `--cream`을 기본으로 유지한다.
- 중요한 CTA는 `--brown` 배경과 흰색 글자를 사용한다.
- 밝은 노란색은 강조에만 사용하며 긴 본문 배경으로 남용하지 않는다.
- 신규 페이지에서도 새로운 색을 무분별하게 추가하지 않는다.
- 새 색상이 필요하면 기존 브라운·옐로 계열과 조화를 확인하고 CSS 변수로 등록한다.

## 5. 타이포그래피

### 사용 폰트

- 제목·로고: `Jua`
- 본문·버튼·설명: `Gowun Dodum`
- 대체 폰트: `system-ui`, `sans-serif`

### 계층 구조

| 요소 | 권장 폰트 | 특징 |
|---|---|---|
| 로고 | Jua | 친근하고 둥근 인상 |
| Hero 제목 | Jua | 가장 강한 시각적 우선순위 |
| 섹션 제목 | Jua | 각 콘텐츠 단위 구분 |
| 카드 제목 | Jua | 짧고 명확하게 작성 |
| 본문 | Gowun Dodum | 가독성과 부드러운 분위기 |
| 버튼 | Gowun Dodum | 일관된 UI 경험 |

### 작성 규칙

- 한국어 제목에는 `word-break: keep-all`을 우선 적용한다.
- 한 문단이 지나치게 길어지지 않도록 한다.
- 버튼 문구는 행동 중심으로 작성한다.
  - 좋은 예: `쿼카 만나기`, `행복 공유하기`, `굿즈 아이디어 보기`
  - 피할 예: `확인`, `클릭`, `다음`

## 6. 레이아웃 시스템

### 컨테이너

```css
--container: 1120px;
```

기본 콘텐츠 너비는 다음 규칙을 사용한다.

```css
.page {
  width: min(var(--container), calc(100% - 40px));
  margin: 0 auto;
}
```

### 여백

- 섹션 상하 여백은 `clamp()`를 사용해 화면 크기에 따라 자연스럽게 조정한다.
- 카드 안쪽 여백은 일반적으로 `24px` 이상을 유지한다.
- 모바일에서는 좌우 여백을 줄이되 콘텐츠가 화면 가장자리에 붙지 않도록 한다.

### 모서리

| 토큰 | 값 | 대표 용도 |
|---|---:|---|
| `--radius-xl` | `36px` | 큰 CTA 박스 |
| `--radius-lg` | `28px` | 소개 박스 |
| `--radius-md` | `20px` | 일반 카드 |

### 그림자

- `--shadow-soft`: 일반 카드와 작은 UI
- `--shadow-strong`: Hero 카드, 모바일 메뉴, CTA 등 시각적 우선순위가 높은 요소

그림자는 깊이를 표현하되 과도하게 진하지 않게 유지한다.

## 7. 페이지 구조

### 7.1 Header

구성:

- 로고
- 소개 링크
- 매력 링크
- 함께하기 링크
- 모바일 햄버거 메뉴

특징:

- 상단 고정형 `sticky` 헤더
- 반투명 크림 배경과 `backdrop-filter`
- 모바일에서 메뉴 버튼을 눌러 내비게이션을 열고 닫음

### 7.2 Hero

구성:

- 브랜드 배지
- 메인 제목
- 프로젝트 설명
- 기본 CTA 2개
- 쿼카 이미지 카드
- 장식용 원형 배경과 말풍선

목표:

- 첫 화면에서 브랜드와 쿼카를 즉시 인식하게 한다.
- 주 CTA는 쿼카 소개 섹션으로 스크롤한다.
- 보조 CTA는 Web Share API 또는 클립보드를 사용한다.

### 7.3 Intro

구성:

- 섹션 라벨과 제목
- 노란색 인용 박스
- 쿼카의 매력과 보호 메시지
- 해시태그 형태의 미니 태그

목표:

- 감성적 메시지와 설명형 콘텐츠의 균형을 맞춘다.

### 7.4 Charm Cards

현재 5개의 카드로 구성된다.

1. 웃는 듯한 얼굴
2. 작고 동글동글함
3. 행복의 상징
4. 보호해야 할 가치
5. 글로벌 확장성

카드 규칙:

- 아이콘 → 제목 → 설명 순서를 유지한다.
- 제목은 한눈에 의미가 전달되도록 작성한다.
- 카드 전체 높이는 데스크톱에서 가능한 한 균일하게 보이도록 한다.
- Hover 시 위로 살짝 이동하고 그림자가 강해진다.

### 7.5 CTA

구성:

- 캠페인 참여 버튼
- 굿즈 아이디어 페이지 이동 버튼

현재 동작:

- 캠페인 버튼: Toast 메시지 표시
- 굿즈 버튼: `./goods.html`로 이동

향후 캠페인 페이지가 만들어지면 Toast 대신 실제 링크 또는 섹션 이동으로 교체한다.

### 7.6 Footer

- 저작권 문구
- 브랜드 무드 키워드

푸터는 정보량을 최소화하고 전체 페이지의 따뜻한 분위기를 유지한다.

## 8. 컴포넌트 규칙

### 버튼

공통 클래스:

```html
<button class="btn">버튼</button>
```

종류:

- `.btn-primary`: 핵심 행동
- `.btn-soft`: 밝은 보조 행동
- `.btn-light`: 진한 배경 위 흰색 버튼
- `.btn-outline`: 진한 배경 위 외곽선 버튼

규칙:

- 기본 높이는 최소 `50px`을 유지한다.
- 둥근 pill 형태를 유지한다.
- 모바일 CTA 버튼은 가능한 경우 전체 너비를 사용한다.
- 클릭 가능한 요소는 키보드로도 접근할 수 있어야 한다.

### 카드

- 배경은 흰색을 기본으로 한다.
- 테두리는 아주 옅은 브라운 계열을 사용한다.
- 제목에는 브랜드 폰트와 메인 브라운을 사용한다.
- 카드 안의 설명은 한 가지 핵심 내용만 전달한다.

### Toast

- 사용자 행동 결과를 짧게 알린다.
- 화면 하단 중앙에 표시한다.
- 기본 표시 시간은 약 `1.8초`다.
- 긴 문장이나 중요한 오류 메시지에는 사용하지 않는다.

## 9. 이미지 스타일

### Hero 이미지

현재 쿼카 이미지는 외부 URL을 사용한다.

적용 효과:

- `object-fit: cover`
- 따뜻한 색감을 위한 `sepia`
- 낮은 채도와 대비
- 가장자리 마스크 효과
- 옐로·브라운 그라디언트 오버레이

### 이미지 운영 원칙

- 운영 배포 시 외부 블로그 이미지 직접 연결보다 프로젝트 내부 에셋 사용을 권장한다.
- 이미지 파일은 가능하면 WebP 또는 최적화된 JPG를 사용한다.
- 의미 있는 이미지에는 구체적인 `alt` 문구를 작성한다.
- 장식용 이미지는 스크린 리더가 읽지 않도록 처리한다.
- 쿼카의 실제 생태나 행동을 설명할 때는 이미지와 설명이 사실과 어긋나지 않도록 검토한다.

## 10. 인터랙션

### 모바일 메뉴

- 버튼 클릭 시 `.open`, `.active` 클래스를 토글한다.
- `aria-expanded` 값을 실제 상태와 동기화한다.
- 메뉴 항목을 선택하면 자동으로 닫는다.
- 화면이 `700px`보다 커지면 모바일 메뉴 상태를 초기화한다.

### 공유 버튼

1. `navigator.share`를 지원하면 기본 공유 UI를 연다.
2. 지원하지 않으면 공유 문구를 클립보드에 복사한다.
3. 취소 또는 오류가 발생하면 Toast로 결과를 알린다.

### 애니메이션

- 원형 배경: 부드러운 상하 이동
- 말풍선: 작은 이동과 확대·축소
- 카드·버튼: Hover 시 위로 이동

`prefers-reduced-motion: reduce` 환경에서는 애니메이션과 전환 효과를 제거한다.

## 11. 반응형 규칙

### Desktop: `1025px 이상`

- Hero는 2열 구조다.
- 매력 카드는 5열로 표시한다.
- 내비게이션 링크를 항상 보여준다.

### Tablet: `1024px 이하`

- Hero를 1열로 전환한다.
- Hero 이미지를 텍스트보다 먼저 보여준다.
- 매력 카드는 2열로 전환한다.
- 소개 박스는 1열로 전환한다.

### Mobile: `700px 이하`

- 햄버거 메뉴를 사용한다.
- 주요 버튼은 전체 너비를 사용한다.
- 매력 카드는 1열로 표시한다.
- CTA와 카드의 모서리 및 여백을 축소한다.

### Small Mobile: `430px 이하`

- 페이지 좌우 여백을 더 줄인다.
- Hero 카드와 이미지 크기를 축소한다.
- 말풍선 위치를 화면 안쪽으로 조정한다.

## 12. 접근성 기준

- 모든 이미지에 목적에 맞는 `alt`를 제공한다.
- 아이콘만 있는 버튼에는 `aria-label`을 제공한다.
- 메뉴 버튼의 `aria-expanded` 상태를 정확히 관리한다.
- 링크와 버튼은 최소 약 `44px`의 터치 영역을 확보한다.
- 텍스트와 배경의 명도 대비를 확인한다.
- Hover에만 의존하지 않고 키보드 Focus 상태도 제공한다.
- 새 컴포넌트를 추가할 때 `:focus-visible` 스타일을 함께 작성한다.
- 움직임 감소 설정을 존중한다.

권장 추가 스타일:

```css
:focus-visible {
  outline: 3px solid var(--yellow);
  outline-offset: 3px;
}
```

## 13. 후속 페이지 확장 원칙

`goods.html`과 같은 후속 페이지는 다음 요소를 공유해야 한다.

- 동일한 로고와 헤더
- 동일한 컬러 토큰
- 동일한 폰트
- 동일한 버튼 모양
- 동일한 컨테이너 너비
- 동일한 카드 모서리와 그림자 강도
- 크림색 페이지 배경

굿즈 페이지에서는 카드형 그리드를 사용하고, 상품 이미지·상품명·짧은 설명을 일정한 순서로 배치한다. 메인 페이지보다 정보량이 많더라도 브랜드의 여백감과 부드러운 분위기를 유지한다.
